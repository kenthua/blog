--- author: Kent Hua blogger_id: tag:blogger.com,1999:blog-3324719914062696225.post-6166866442132550154 blogger_orig_url: http://blog.kenthua.com/2015/07/openshift-enterprise-30-all-in-one.html comments: true date: "2015-07-21T16:25:00Z" modified_time: "2015-08-21T10:46:27.462-07:00" title: 'OpenShift Enterprise 3.0 all in one vagrant build ' --- It's been a while since I last posted, but here is something different.  I spent sometime playing with vagrant and for my first attempt it took a while to get used to.

However, the result is a working vagrant build which will build out an all in one OSE environment.  There is one external dependency which is an external DNS server.  I think I will eventually make a change so you can just have it point to an external domain of yours.
It works, however, I'm sure there are better ways or efficient ways to make things better.  If there are some tips/suggestions please send them my way.

Lots of dependencies of course, such as a Red Hat OpenShift Enterprise subscription.

So where to get it?  Github of course.  Link here.

Do you want to just do everything with the upstream code or you don't have a subscription?  The openshift origin team put together a development vagrant build on github as well.
